# Проект: Детекция персонажей в Minecraft с использованием FCOS и YOLO

## 1. Назначение и цель проекта

Исследовать возможности современных моделей детекции объектов на примере игрового мира Minecraft. Выполнить дообучение детекторов на специфическом игровом датасете и сравнить их по точности, скорости и качеству предсказаний.

**Цель проекта:**
- Дообучить две модели детекции: **FCOS** и **YOLO**.
- Провести инференс моделей на видеопотоке для визуальной оценки.
- Сравнить модели по метрикам **mAP** и **FPS**.

## 4. Этапы выполнения

### 4.1. Исследовательский анализ данных (EDA)

- Проверить структуру датасета (формат COCO).
- Перенести файлы аннотаций в `datasets/minecraft/annotations/`.
- Проверить корректность JSON-структуры.
- Сопоставить количество изображений и аннотаций.
- Визуализировать распределение классов, выявить дисбаланс.
- Визуализировать одно тестовое изображение с bounding box'ами и подписями классов.
- Сформулировать план выполнения проекта.

### 4.2. Настройка конфигурации FCOS

Создать файл `configs/fcos/fcos_minecraft.py` с параметрами:

| Параметр | Значение |
|----------|----------|
| `_base_` | базовый конфиг FCOS (например, `fcos_r50_caffe_fpn_gn-head_1x_coco.py`) |
| `metainfo` | 17 классов Minecraft |
| `img_scale` | (512, 512) — сжатие для ускорения |
| `max_epochs` | не более 12 |
| `samples_per_gpu` | 2 |
| `workers_per_gpu` | 2 |
| `checkpoint_interval` | 1 (сохранение раз в эпоху) |
| `fp16` | `dict(loss_scale='dynamic')` — половинная точность |

**Аугментации:** базовые (без Mosaic и тяжёлых методов).

### 4.3. Настройка YOLO

- Создать файл `datasets/minecraft/data.yaml` с указанием:
  - пути к train/val/test;
  - список классов (17).
- Использовать **YOLOv8s** как базовую модель.

### 4.4. Тестирование предобученных моделей (инференс)

**FCOS:**
- Скачать чекпоинт из официального репозитория в `checkpoints/`.
- Выполнить инференс на тестовом изображении.
- Сохранить результат в `artifacts/inference/test_pretrained.jpg`.

**YOLO:**
- Загрузить предобученную YOLOv8s.
- Выполнить инференс на тестовом изображении.
- Сохранить результат в `artifacts/inference/yolo_val/`.

### 4.5. Обучение моделей

**FCOS:**
- Запустить дообучение на train-датасете.
- Сохранить логи и веса в `artifacts/fcos/`.
- Визуализировать метрики из `log.json`.

**YOLO:**
- Запустить дообучение (допускается больше эпох и другие аугментации).
- Сохранить логи и веса в `artifacts/yolo/`.
- Визуализировать метрики из `results.csv`.

### 4.6. Инференс на тестовых изображениях и видео

**Изображения:**
- Прогнать обученные модели на тестовых изображениях.
- Сохранить результаты:
  - FCOS → `artifacts/inference/fcos/`
  - YOLO → `artifacts/inference/yolo/`
- Визуализировать по 5 изображений в ноутбуке для каждой модели.

**Видео:**
- Выполнить инференс на `datasets/minecraft/video.mp4`.
- Сохранить результаты:
  - FCOS → `artifacts/videos/fcos_inference.mp4`
  - YOLO → `artifacts/videos/yolo_inference.mp4`

### 4.7. Сравнение метрик

Рассчитать для каждой модели на тестовых изображениях:

| Метрика | Описание |
|---------|----------|
| `mAP` | средняя точность (mean Average Precision) |
| `mAP_50` | mAP при IoU = 0.5 |
| `FPS` | кадров в секунду (скорость инференса) |

Вывести таблицу сравнения. Сохранить в `artifacts/metrics/metrics_comparison.csv`.

### 4.8. Анализ и выводы

Подготовить отчёт `artifacts/report.pdf`, содержащий:

- Выводы о моделях на основе метрик качества (mAP, mAP_50).
- Выводы о моделях на основе скорости инференса (FPS).
- Примеры результатов детекций.
- Графики для визуального сравнения.
- Краткое описание проекта в `README.md`.

## 7. Примечания

- Размер изображения сжимается до (512, 512) для ускорения обучения.
- Для FCOS используется половинная точность (`fp16`).
- YOLO обучается быстрее, допускается больше эпох.
- Видео для инференса скачивается отдельно и размещается в `datasets/minecraft/video.mp4`.
- Сами данные (датасет) в репозиторий не загружаются, только структура и пути.
